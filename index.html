⚡ Why We Use Shopify CLI

Before touching any code, every developer on the team needs to understand what Shopify CLI is, why it exists, and what problem it solves.

⚡
The core idea

Shopify stores run in the cloud. Normally, to edit a theme you'd have to go into the Shopify Admin, open the online code editor, and edit files one by one in a browser. Shopify CLI fixes this — it connects your local VS Code editor directly to your live store, so you can use proper developer tools and see changes instantly.

❌ Without CLI
  • ✗Edit code inside Shopify's browser editor — slow and limited
  • ✗No autocomplete, no syntax highlighting, no extensions
  • ✗Save file → wait for page reload every time
  • ✗Can't use AI tools or Claude Code
  • ✗Every change is immediately on the live theme — no safe testing
  • ✗No Git, no version control, no team collaboration
✅ With Shopify CLI
  • ✓Code in VS Code — full extensions, AI, IntelliSense
  • ✓Hot reload — save a file, browser updates instantly
  • ✓Preview URL — only you see your changes, not customers
  • ✓Claude Code MCP reads your files — AI understands your project
  • ✓Works with Git — full version control and team collaboration
  • ✓Multiple devs can work on the same project simultaneously

4 things every developer must understand about CLI
👁️
Preview Theme
When you run shopify theme dev, CLI creates a private preview theme on the store just for you. Your changes are 100% invisible to real customers until you choose to deploy.
🔥
Hot Reload
Every time you save a .liquid, .css, or .js file, the browser refreshes automatically. No manual upload. No page refresh needed. Changes appear in under a second.
🏪
Real Store Data
Unlike a fake local environment, CLI connects to your real store — real products, real collections, real metafields. What you see in preview is exactly what customers will see after deploy.
📂
Local Files = Source of Truth
Your local project folder is the master copy of the theme. The Shopify store is just the output. Always edit locally in VS Code — never edit directly in Shopify Admin's code editor.
⚠️ What CLI alone cannot do — this is where Git comes in
→
Track who changed what
CLI has no history. If something breaks you can't see who did it or when.
→
Multiple devs at once
CLI is single-user. Two devs pushing at the same time will overwrite each other.
→
Backup & rollback
CLI has no undo. Push bad code to live? You're stuck without Git history.
→
Code review before deploy
CLI pushes straight to the store. No approval step, no second set of eyes.
💡 Think of it this way: Shopify CLI is your connection to the store. Git is your safety net and team coordination. You need both. CLI without Git = powerful but dangerous. Git without CLI = version control but no live preview. Together = the professional workflow.

terminal

🛡️Before your first pull, make sure a .shopifyignore file exists in your project root. It prevents Shopify from downloading unnecessary files (like node_modules/, .git/, config secrets) that should never be pulled or pushed to the store.

⚠️ Why You MUST Use Git

Using Shopify CLI alone is dangerous for a team. Here's exactly what goes wrong — and how Git fixes every problem.

🚨 The CLI-Only Problem

When you run shopify theme push without Git, Shopify lists your themes like this: first is always your LIVE theme. One wrong selection and your live store is overwritten instantly — with no backup, no history, and no idea who did it.

❌ Without Git — What actually happens

→

→

No backup. No history.

✅ With Git — Safe, trackable, recoverable

Dev A works on branch
→
Commits with name
→
PR reviewed
→
Merge → main
→
Auto deploy ✅
🔍
Track Every Change
Every commit shows who changed what and when. No more "who broke the homepage?" — git log tells you instantly.
👥
Multiple Devs, Zero Conflicts
Frontend and backend devs each work on their own branch. They never overwrite each other. Merge only when ready and reviewed.
💾
Always Have a Backup
Every version of your theme is stored in GitHub history. If something breaks on live, roll back to any previous commit in seconds.
🛡️
Never Touch Live by Mistake
Only merges to main trigger a live deploy. Feature branches deploy to a preview theme — so devs can test safely.
🚨 Rule #1: Nobody pushes directly to main. Nobody runs shopify theme push --publish manually. All changes go through a branch → review → merge workflow. No exceptions.

# Shopify Theme — AI Instructions

## Project
Shopify 2.0 theme. Store: your-store.myshopify.com

## Stack
- Shopify Liquid + JSON templates (sections everywhere)
- Vanilla JS only (no jQuery, no frameworks)
- CSS custom properties (see assets/base.css for variables)
- Shopify CLI 3.x for local development

## Code Rules
- Every section MUST have a {% schema %} block
- CSS must be scoped: #shopify-section-{{ section.id }} { }
- All images: use Shopify image_url filter + lazy loading
- No inline styles. No hardcoded text. No hardcoded colors.
- Accessible: all buttons need aria-labels, images need alt text
- Mobile-first CSS always

## File Locations
- Sections: /sections/*.liquid
- Reusable partials: /snippets/*.liquid  
- Global styles: /assets/base.css
- Global JS: /assets/theme.js

## CSS Variables (from base.css)
--color-base-text, --color-base-background-1,
--color-base-accent-1, --font-body-family, --font-heading-family

## When Creating Sections
1. Look at existing sections for patterns
2. Use existing CSS variables, don't invent new ones
3. Match the code style of the project

📋 AI Coding Rules & Careful Tips

What to always tell the AI, what to never let it do, and what to always check before committing AI-generated code.

✅ Always Tell the AI

  • Use Shopify 2.0 sections-everywhere architecture
  • Add complete {% schema %} to every section
  • CSS scoped to #shopify-section-{{ section.id }}
  • Use Shopify image_url filter for all images
  • Make all text & colors configurable via schema
  • Mobile-first CSS always
  • Vanilla JS only — no jQuery, no libraries
  • Add aria-labels and alt text for accessibility

❌ Never Let the AI Do

  • Hardcode store URLs, product names, or prices
  • Use jQuery or any CDN library without asking
  • Write inline styles instead of CSS classes
  • Skip lazy loading on images
  • Create sections without schema blocks
  • Use window.location for store URLs
  • Ignore mobile layouts
  • Commit directly to main (AI can't do this anyway)
⚠️ Careful Tips — Read Before Using AI Code
👁️Always review AI code before committing. AI is great at structure but sometimes invents Shopify filters that don't exist, or uses deprecated Liquid syntax. Always test with shopify theme dev first.
🔢Don't paste your entire theme into AI chat. Use Claude Code MCP instead — it reads files directly. Pasting huge files wastes tokens and confuses the model.
🧪Test AI-generated sections on mobile. AI often writes desktop-first CSS. Always check the preview URL on a phone before opening a PR.
🔑Never paste secrets, passwords, or API tokens into AI chat. Not Claude, not Copilot, not ChatGPT. If you need the AI to work with credentials, use environment variables and tell it the variable name only.
💡AI is not a replacement for code review. Every AI-written feature still needs a PR and a teammate to review it. Think of Claude Code as a very fast junior developer — good but needs checking.
